<table> tag :
- For creating tables
- <table border> <tr>  <th colspan="2">NAME</th> <td rowspan="3">Aa</td>  </tr> </table>
- Inside table we have rows(<tr>- table rows). 
- Inside rows we have
     column header(<th>- table header cell) (Bold)
                 or
     column data(<td>- table data) 

- <tr> acts as rows
- <td> or <th> acts as columns and cell data!!
     Use <th> for column name (bold)
     Use <td> for normal cell data

 attributes:
- <table> attribute like "border." to put all borders, "cellpadding", "cellspacing", maybe more
- <th> or <td> attribute like "colspan" and "rowspan" , tells #rows to span and #columns to span respctively.

*Remember: a table-data or table-header can have images, videos, anything. Not only Text.

Forms <form> tag :               "Interactive Web-Pages"

- like a gateway for 'users' to provide information like signing up, searching or completing a transaction
- hence makes websites dynamic and interactive
- Inside form-tag we have 3 elements normally: (form elements)
     -label &
     -mechanism to accept input (input)     -typed input
     -buttons                             -clicked input

- <form> attribute "action"
- <input /> tag to accept data from the user.
- <input> tag attributes like - "type"(mandatory), "name", "autofocus.", "placeholder", "required.", "size", "max"-"min"-"step" attributes for number type(with SPIN CONTROL) and many more......
- "type" attribute values like - 'text', 'number', 'submit', 'reset'
- "name" attribute crutial while form submission since it associates value given by user with it (un=JohnDoe) (u can see that in the url while submitting!!)
- <label> tag attribute "for"(optional) to specify 'name or id' of "input" element it belongs to

note: the . placed by 'me' after attribute names is to make understand that no value needed to that attribute ! (like singleton attribute!!) i named it

- "size" to adjust width of the input box (Only applicable to some input types. not all)
- "id"
- "title" attribute to show text when hovered on box (like showing user- password shd be 6 char long)

some more <input /> 'types' (form elements):
- radio    (choose only 1 option)
- checkbox    (choose multiple options)
- button

- Usually radio type will have multiple options to choose from
- For radio type to work, 'same name' should be given to all the options  (logically makes sense), otherwise all the options will be selectable !!!! (which is contradictory to radio button) 
- In (checkbox, radio and range) <input /> types, "value" attribute acts as the value to sent to the server(request-string) with "name". Ex: if value="hi" then 'name=hi' is sent in request string
- "checked" attribute for default check option
- if "value" attribute not specified in 'radio' and 'checkbox' type , no value is sent in the request string!!! . So specify values
- In 'button' <input /> type,  attribute "value" acts 'differently'. It gives button-name inside button!!
- 'range' type in input tag also has attributes like "min", "max", "value"... (value also acts as default value b/w min and max)
- in 'text' input type "value" acts as DEFAULT text 


- more attributes summary: "value", "checked."

- button - another type for input tag. Functionality to this given using js or other languages
- if "value" not specified to it, button looks small w/o any name on it. So give value

- we also have <button> tag itself (still not seen)
